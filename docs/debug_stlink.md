# 用 ST-Link 调试：看变量 / 改变量（不需要按键）

你问的核心点是两件事：
1) “是不是还要烧录？”  
2) “怎么在程序跑着的时候看/改 us 变量？”

## 结论（先讲清楚）

- 第一次必须“下载/烧录”一次带调试信息的固件（Debug 配置通常就带）。
- 之后你可以用 ST-Link 进入 Debug 会话，查看变量；很多情况下也可以直接改 `volatile` 变量的值，不需要每改一次就重新烧录。

注意：进入调试会话时，CPU 常常会先暂停（halt），你需要点一次 Resume 让它继续跑。

## CubeIDE 操作步骤（通用）

1) 工程选择 `Debug` 配置编译（确保有符号信息）。
2) `Run -> Debug` 启动调试（或工具栏小虫子图标）。
3) 如果程序停住不跑：
   - 点 `Resume(F8)` 继续运行
4) 打开变量观察窗口：
   - `Window -> Show View -> Expressions`
   - 在 Expressions 里添加你要看的变量（例如 `g_arm.last_pulse_us[0]` 或 `g_calib_us`）

## 让变量“能被改/能看见”的小技巧

- 用 `volatile` 修饰：避免编译器优化导致变量在调试器里不好改/不好看。
- 不要把变量写成函数内的临时变量；建议放到文件作用域（全局/static）或结构体里。

## 连接不上/一连就跑飞的常见原因

- 供电不稳（舵机动作导致复位）
- ST-Link 供电给板子（不推荐）
- 没接 `GND` 共地

排查顺序建议：
- 先断开所有舵机，只保留板子 + PCA9685 逻辑供电 + I2C，看是否稳定连接 Debug
- 再逐个加回舵机电源与舵机
